## ğŸš€ **Getting Started Tutorial**

### ğŸ“‹ **Prerequisites**

Before you begin, make sure you have the following installed on your machine:

- **Node.js** (version 18 or higher) - [Download here](https://nodejs.org/)
- **npm** or **yarn** package manager
- **Git** - [Download here](https://git-scm.com/)
- **Google account** for Firebase setup
- **Code editor** (VS Code recommended)

### ğŸ”§ **Step-by-Step Installation**

#### **Step 1: Clone the Repository**

\`\`\`bash

# Clone the repository

git clone https://github.com/ricard027/Flugo

# Navigate to the project directory

cd flugo-employee-management

# Check if you're in the right directory

ls -la
\`\`\`

#### **Step 2: Install Dependencies**

\`\`\`bash

# Using npm

npm install

# OR using yarn

yarn install

# This will install all required packages including:

# - Next.js, React, TypeScript

# - Material-UI components

# - Firebase SDK

# - And all other dependencies

\`\`\`

#### **Step 3: Firebase Setup (Important!)**

1. **Create a Firebase Project**

   - Go to [Firebase Console](https://console.firebase.google.com/)
   - Click "Create a project"
   - Enter project name (e.g., "flugo-employee-management")
   - Enable Google Analytics (optional)
   - Click "Create project"

2. **Enable Authentication**

   - In Firebase Console, go to "Authentication"
   - Click "Get started"
   - Go to "Sign-in method" tab
   - Enable "Google" provider
   - Add your domain to authorized domains

3. **Create Firestore Database**

   - Go to "Firestore Database"
   - Click "Create database"
   - Choose "Start in test mode" (for development)
   - Select a location close to your users

4. **Get Firebase Configuration**
   - Go to Project Settings (gear icon)
   - Scroll down to "Your apps"
   - Click "Web app" icon (</>)
   - Register your app with a name
   - Copy the configuration object

#### **Step 4: Environment Variables Setup**

\`\`\`bash

# Copy the example environment file

cp .env.local.example .env.local

# Open the file in your editor

code .env.local
\`\`\`

Fill in your Firebase configuration in `.env.local`:

\`\`\`env

# Firebase Configuration

NEXT_PUBLIC_FIREBASE_API_KEY=your_api_key_here
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your_project_id_here
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=123456789
NEXT_PUBLIC_FIREBASE_APP_ID=1:123456789:web:abcdef123456
NEXT_PUBLIC_FIREBASE_MEASUREMENT_ID=G-XXXXXXXXXX
\`\`\`

#### **Step 5: Configure Firestore Security Rules**

In Firebase Console, go to Firestore Database > Rules and replace with:

\`\`\`javascript
rules_version = '2';
service cloud.firestore {
match /databases/{database}/documents {
// Allow authenticated users to read/write employees
match /employees/{document} {
allow read, write: if request.auth != null;
}
}
}
\`\`\`

Click "Publish" to save the rules.

#### **Step 6: Run the Development Server**

\`\`\`bash

# Start the development server

npm run dev

# OR with yarn

yarn dev
\`\`\`

You should see output like:
\`\`\`
âœ“ Ready in 2.3s
âœ“ Local: http://localhost:3000
âœ“ Network: http://192.168.1.100:3000
\`\`\`

#### **Step 7: Open Your Browser**

1. Open your browser and go to `http://localhost:3000`
2. You should see the Flugo landing page
3. Click "Entrar" (Login) to test Google authentication
4. After login, you'll be redirected to the employee list

#### **Step 8: Populate Sample Data (Optional)**

1. After logging in, go to the "Colaboradores" page
2. If the table is empty, click "Popular Dados" button
3. This will add sample employee data to test the system

## ğŸ› **Troubleshooting**

### **Common Issues and Solutions**

#### **1. Firebase Authentication Error**

\`\`\`
Error: Firebase configuration not found
\`\`\`
**Solution:** Make sure your `.env.local` file has all Firebase configuration variables.

#### **2. Permission Denied Error**

\`\`\`
Error: Missing or insufficient permissions
\`\`\`
**Solution:** Check your Firestore security rules and make sure authentication is working.

#### **3. Module Not Found Error**

\`\`\`
Error: Cannot resolve module
\`\`\`
**Solution:** Delete `node_modules` and reinstall:
\`\`\`bash
rm -rf node_modules package-lock.json
npm install
\`\`\`

#### **4. Port Already in Use**

\`\`\`
Error: Port 3000 is already in use
\`\`\`
**Solution:** Use a different port:
\`\`\`bash
npm run dev -- -p 3001
\`\`\`

#### **5. Google Login Popup Blocked**

**Solution:** Allow popups for localhost in your browser settings.
